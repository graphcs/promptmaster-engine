# AI UI Generator Prompt

Use this single-paragraph prompt when generating screen mockups in AI UI tools (v0, Galileo AI, Uizard, etc.):

---

Design a complete set of screens for PromptMaster Engine, a professional SaaS web application that structures interactions with AI language models using a 5-phase guided workflow — it is NOT a chatbot or conversational AI tool, it is a structured workflow platform for enterprise professionals like analysts, auditors, lawyers, and strategists who need aligned, evaluated AI outputs. The application uses a fixed sidebar layout (260px sidebar on the left with session management, model selector, tier badge, and navigation; main content area on the right with max-width 720px) built with Next.js, Tailwind CSS, and shadcn/ui components. The design system uses Inter font, a trust-blue primary color (#2563EB), slate-50 background (#F8FAFC), white cards with subtle shadows, 8px border-radius buttons, 12px border-radius cards, and Lucide icons throughout — the aesthetic should feel like Linear or Vercel's dashboard: clean, spacious, confident, with no playful elements or AI gradients. The 5 screens to generate are: (1) Input Phase — the main workspace showing a grid of 8 clickable mode selection cards at the top (Architect, Critic, Clarity, Coach, Therapist, Cold Critic, Analyst, Custom — each with an icon and one-line tagline, with the selected card highlighted in blue with an expanded inline "About this mode" description panel below it), followed by an objective textarea, audience dropdown, and two columns of preset constraint chips (toggleable pill buttons like "Keep it under 300 words", "No jargon") above editable textareas for constraints and output format, ending with a prominent "Assemble Prompt" primary button; (2) Review Phase — showing a read-only system prompt in a collapsible section, a large editable user prompt textarea, a scaffolding toggle checkbox, and "Back to Input" / "Execute" buttons; (3) Output & Evaluation Phase — displaying the AI-generated output rendered as formatted markdown, followed by an evaluation card split into two sections: "Quality Scores" showing Alignment and Clarity badges in a 2-column grid (each badge is a colored pill — green for High, amber for Medium, red for Low — with a one-sentence explanation below), and "Scope Check" showing Drift as a full-width row with an explanatory caption "Drift measures whether the output stayed focused on your objective", followed by an LLM-generated "Suggestions" section with 2-3 contextual bullet points, then conditional action buttons (either "Generate Realignment Prompt" / "Refine Prompt" / "Proceed Anyway" if scores are poor, or "Finalize Session" / "Refine Prompt" if scores are good), plus a download button and collapsible iteration history with trend arrows (▲ improved, ▼ declined); (4) Realignment Phase — an info banner explaining the realignment process, a large editable textarea pre-filled with the corrective prompt, and "Back to Output" / "Execute Realignment" buttons; (5) Summary Phase — the final session view showing evaluation scores at top, three metric cards (Mode, Iterations, Final Alignment), the final output rendered as markdown, export buttons (Download Summary .txt and Download Session .json), a collapsible iteration comparison view with side-by-side dropdowns, a Self-Audit section with a "Run Self-Audit" button that triggers a Cold Critic analysis, and bottom action buttons "Start New Session" (primary) and "Hard Reset (carry lessons forward)". Additionally, generate a Login page with email/password form, Google OAuth button, and PromptMaster branding, and a phase stepper component that appears in the sidebar showing the 5 phases as connected circles (completed=green checkmark, active=blue filled, upcoming=gray outline).
